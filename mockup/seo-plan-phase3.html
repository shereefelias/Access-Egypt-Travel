Schema Markup

Step 1 — Add Structured Data (Schema Markup) via Rank Math

Why this matters: Schema markup is JSON-LD code that tells Google exactly what type of content a page contains — a FAQ, a business, a tour product, a person. Google uses this to display rich results: FAQ accordions, star ratings, and knowledge panel data. These appear above regular search results and dramatically increase click-through rate without needing to rank higher.
All schema below is added via Rank Math → Schema inside each page's editor. Rank Math generates the JSON-LD automatically — you fill in fields in a form, not code. For copy-paste JSON-LD code, see schema-markup.html.

1A
FAQPage Schema — FAQ Page

This is the fastest visible win. When Google detects FAQPage schema, it shows your FAQ questions as an expandable accordion directly in search results — without you ranking higher. It's free clicks.

Edit FAQ page → Rank Math panel → Schema tab → click + Add Schema → choose FAQ

Rank Math will show a form where you add each question and answer pair. Add all FAQ items. Each question should match exactly what appears on the page — Google cross-references them.

Each answer must be 1–3 sentences max. Google truncates long FAQ answers in the rich result. Make the first sentence the complete answer.
After saving, test it at search.google.com/test/rich-results — paste your FAQ page URL and it will show you whether Google can read the schema correctly.

Image SEO

Step 2 — Image SEO: Alt Text, og:image, and Discover Eligibility

Why this matters: Google cannot see images — it reads alt text and metadata to understand them. Missing alt text means your images contribute zero SEO value. Missing og:image means your pages won't get large preview images in Google Discover (the mobile interest feed). These are easy fixes with measurable impact.
2A
Alt Text — Run the Python Script Already in Your Folder

You have generate_alt_text.py in your project folder that can auto-generate alt text for all images in your WordPress Media Library using Claude AI. This needs to be run after your developer completes Phase 1 and all real images are uploaded.

Terminal → python3 generate_alt_text.py

After generating alt text, spot-check these rules manually:

  • ✓ Descriptive: "Karnak Temple hypostyle hall columns at sunrise, Luxor Egypt"
  • ✓ Natural language, not keyword-stuffed
  • ✗ Not: "egypt tour luxor temple egyptologist private tour guide"
  • ✗ Not: empty / "image" / "photo" / "IMG_4392"
  • Decorative images (icons, dividers): use alt="" — empty on purpose
Elementor sometimes uses CSS background images for hero sections. Google cannot index CSS backgrounds — they contribute zero SEO value for images. Ask your developer to check whether hero images are <img> tags or CSS backgrounds. If they're CSS backgrounds, they need to be converted to Elementor's Image widget or Background Image field (which outputs proper img tags in newer Elementor versions).
2B
Descriptive Filenames Before Upload

Rename image files before uploading to WordPress. Google reads filenames as an additional signal.

❌ Bad (don't upload these)✅ Good (rename first)
IMG_4392.jpgvalley-of-the-kings-luxor-egypt.jpg
DSC00234.jpgkarnak-temple-columns-sunrise.jpg
photo.jpgjohn-gamil-egyptologist-guide-giza.jpg
tour1.jpgnile-cruise-luxor-aswan-private-tour.jpg
Use lowercase, hyphens (not underscores or spaces), descriptive terms. You cannot rename files after upload in WordPress without a plugin — do this before uploading.
2C
og:image — Set a Feature Image on Every Page

The og:image tag tells Google (and social media) which image to use when sharing or featuring your page. Without it, Google picks randomly — often your logo or nothing.

Edit any page → Rank Math panel → Social tab → set Facebook Image

Rank Math outputs this as <meta property="og:image"> automatically. Set this on every page and blog post.

The image you set here is what appears when your page shows up in Google Discover (the mobile interest feed). Choose a compelling, high-quality photo — not the logo.
2D
Enable max-image-preview:large — Required for Google Discover

Without this meta tag, Google won't show your pages with large preview images in Discover, even if your images are high-quality. It's one line of code, site-wide.

Rank Math → Titles & Meta → Global Meta → find Max Image Preview → set to Large

This outputs <meta name="robots" content="max-image-preview:large"> on every page. Alternatively, add it manually to your theme's <head> via Appearance → Theme File Editor → header.php.

Discover image requirements: Images must also be at least 1200px wide and 300,000+ total pixels (e.g., 1280×720 = fine). Low-resolution images will not get large Discover previews even with this tag set.
2E
Verify Image Sitemap Is Active

Rank Math includes image URLs in your XML sitemap by default. Verify it's working:

https://accessegypttravel.com/sitemap_index.xml

Open that URL and look for a sitemap entry that includes image data. If images aren't in the sitemap, go to Rank Math → Sitemap → Modules and ensure Images is enabled.

Core Web Vitals

Step 3 — Core Web Vitals: Google's Performance Ranking Signal

Why this matters: Core Web Vitals (CWV) are Google's official page experience signals used as a ranking factor. A site that fails CWV is at a disadvantage against competitors that pass, even if the content is equivalent. Elementor sites commonly fail LCP and CLS out of the box.

The 3 Signals Google Measures

SignalWhat It MeasuresTargetCommon Elementor Issue
LCP
Largest Contentful Paint		 How long until the biggest visible element loads Under 2.5 seconds Large unoptimized hero images, full CSS bundle loading
INP
Interaction to Next Paint		 How fast the page responds to clicks/taps Under 200ms Too many Elementor widgets, unoptimized JavaScript
CLS
Cumulative Layout Shift		 How much elements jump around while loading Under 0.1 Images without explicit width/height, fonts loading late
3A
Measure Your Current Scores First

Run your homepage through both tools:

https://pagespeed.web.dev/
https://search.google.com/search-console → Core Web Vitals report

PageSpeed Insights shows your scores immediately. Search Console shows real-user data over the past 28 days (more accurate, but only available after your site has enough traffic).

Run the test on Mobile — not just Desktop. Google uses mobile-first indexing, so mobile scores are what actually affects rankings.
3B
Fix LCP — Optimize Your Hero Images

The hero image on each page is almost always the LCP element. These are the highest-impact changes:

ActionHow to Do ItImpact
Convert hero images to WebP format Use Smush or ShortPixel WordPress plugin — bulk convert all images to WebP automatically 30–50% smaller file size
Add Fetchpriority="high" to hero image In Elementor, edit the hero image widget → Advanced → add attribute fetchpriority="high" Tells browser to load this image first
Disable lazy load on hero image only Hero image should NOT be lazy-loaded — it's always visible. In Elementor image widget → Advanced → Loading: Eager Prevents unnecessary delay on visible image
Use correctly sized images Don't upload a 4000px image if it displays at 1400px. Resize before upload. Saves bandwidth, speeds load
3C
Fix CLS — Set Explicit Image Dimensions

Layout shift happens when the browser doesn't know how tall an image is until it loads — so text jumps down when the image appears. Fix by always setting explicit width and height on images.

In WordPress, when you upload an image and insert it with the Image widget in Elementor, set both Width and Height values in the widget settings. This tells the browser to reserve space before the image loads.

Also check for font-related CLS: if custom fonts are loading late, they cause text to jump. Use font-display: swap in your theme's CSS to prevent invisible text during font load.
3D
Install a Caching Plugin

No caching plugin is currently in the developer plan. This is a significant gap — caching is the single highest-ROI performance improvement for WordPress sites.

PluginBest ForCost

Ask your developer to install and configure one of these. WP Rocket is the easiest to set up and gets the best results, but any of the free options will help significantly.

E-E-A-T

Step 4 — E-E-A-T Signals on the 6 Content Pages

Why this matters: Google's quality raters evaluate content by asking "who wrote this, and why should I trust them?" For travel content — which affects how people spend significant money — Google applies extra scrutiny (YMYL: Your Money Your Life). The 6 content pages planned in Phase 2 will underperform without E-E-A-T signals, no matter how good the writing is.
The non-commodity test: Before publishing any content page, ask: "Could an AI write this without having been to Egypt?" If yes, it's commodity content — Google's AI systems filter it out when generating AI Overviews. John Gamil's 22 years of on-the-ground experience is your unfair advantage. Every page must use it.

Internal Linking

Step 5 — Internal Linking Strategy

Why this matters: Internal links pass authority (PageRank) from informational pages to money pages (tour pages). A visitor who reads "Is Egypt Safe?" and then sees a link to "Egypt Packages" with a personal CTA is much more likely to convert than one who finishes the article and leaves. Internal links also signal to Google which pages are most important on your site.

Link Map — What Links to What

Is Egypt Safe? → Egypt Packages page ("Browse our private Egypt tours — led by a local who's been doing this for 22 years") + Contact page ("Ask us anything about safety — we're available by phone, WhatsApp, or Calendly")
Hurghada to Luxor → Luxor Tours page ("See our full Luxor day tour options") + Contact page ("Ready to book a Hurghada pickup? Contact us")
Cairo to Luxor → Egypt Packages page ("View packages that include Cairo-to-Luxor transfer") + Nile Cruise page ("Add a Nile cruise to your Cairo-to-Luxor journey")
Best Time to Visit → Tailor-Made Tours page ("Build a custom itinerary around your preferred travel dates") + Contact page
Egypt Visa Guide → FAQ page ("See more Egypt travel questions answered") + Contact page ("We help clients with pre-trip logistics too")
What to Pack → FAQ page + Best Time to Visit page ("Packing depends on when you're going — see our seasonal guide")
FAQ Page → Every tour page (relevant FAQ answers should link to the relevant tour) + all 6 content pages (each FAQ can say "full guide here")
Tour Pages → Related tours ("You might also like: Nile Cruise") + Destination pages for each stop on the tour + Contact/Book page

Rules for Every Internal Link

5A
Use Descriptive Anchor Text — Never "Click Here"
❌ Don't Use✅ Use Instead
"Click here for our tours""Browse our private Egypt tour packages"
"Learn more""Read our complete Hurghada to Luxor guide"
"Contact us""Ask us about pickup from Hurghada"
"See here""View the Nile Cruise itinerary"

Google reads anchor text to understand what the linked page is about. Descriptive anchor text is a free ranking signal for the destination page.
